cmdg - A command line client to GMail

Project home page / description

Copyright Thomas Habets <thomas@habets.se> 2015-2019

cmdg is a commandline client to GMail that provides a UI more similar to Pine/Alpine.

cmdg is open source with the code available on github. See that project page for installation and setup instructions.

It uses the GMail API to interact with your mailbox. This has several benefits.

Benefits over IMAP

  • No passwords stored on disk. (application-specific passwords are also passwords, and can be used for more than GMail). OAuth2 is used instead, and cmdgs access can be revoked here. cmdg can only access your GMail, and cannot lose your password even if the machine it runs on gets hacked.
  • The "labels" model is native in the cmdg UI, unlike IMAP clients that try to map GMail labels onto IMAP.
  • Contacts are taken from your Google contacts

Benefits over the GMail web UI

  • Emacs-ish keys. If there's a need the key mapping can be made configurable.
  • Uses a real $EDITOR.
  • Really fast. No browser, CSS, or javascript getting in the way.
  • Proper quoting. The GMail web UI encourages top-posting. Ugh
  • It uses 100% keyboard navigation. GMail web UI has very good keyboard navigation for a web app, but still requires mouse for a few things
  • cmdg works without a graphic terminal
  • cmdg uses less bandwidth (citation needed), and much less memory
  • Local GPG integration. There are currently no *good* ways to integrate GPG with the GMail web UI

Google OAuth consent screen requires a home page and privacy policy, so this is it.

cmdg Privacy policy

What data cmdg accesses

  • Your emails when you read them
  • Your emails when you write them
  • Your contacts in order to help you send email to them
  • Its own settings in a hidden Google Drive folder

What data cmdg shares

Emails you send with cmdg are sent to your recipients via the GMail API, exactly what you'd expect.

In plain words: This is an email client you run on your own machine as an interface to the GMail API. It won't upload your emails anywhere for "processing" by any third party. Your emails just go between you on your machine and your GMail account.

What data cmdg stores

If cmdg fails to send an email for you (for example due to a network outage), it give you the option to save the draft to your local hard drive.

Settings, such as your email signature, is saved in Google Drive for you in your account, and nobody else is given access to such settings.

cmdg never stores any data outside of your Google account (and doesn't grant access to anyone else), and only sends data anywhere, and then only to the recipient, when you choose to send an email.

The only thing stored locally is the OAuth access tokens, just like how Chrome has access tokens to browse to mail.google.com.

How cmdg uses your data

No ads, no sharing to third parties or to cmdg authors. Your data is only used and processed as required to read and write emails.

Changes to This Privacy Policy

This privacy policy will not substantially change, but if in order to provide a better email client by e.g. using more APIs cmdg processes more user data, it will be added to this page.
